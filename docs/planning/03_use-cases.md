# ユースケース定義

> 詳細なユーザー像は [02_personas.md](./02_personas.md) を参照。
> ユースケースは「MVP対象」と「将来対応」に分けて管理する。

## 基本方針: ログイン必須でMVPをシンプルに保つ

> MVPはログインユーザーの記録CRUD機能に集中する。
> 実装の複雑度を下げ、高いソフトウェア品質を保ちながら早期デリバリーを優先する。
>
> | 状態 | 記録の保存先 | 備考 |
> |------|-------------|------|
> | ログイン済み | Supabase DB（クラウド） | MVP対象 |
> | 未ログイン | localStorage（ブラウザ内） | 将来対応（US-01 / US-08） |
>
> アカウントなしで記録できる機能（US-01）とその記録の引き継ぎ（US-08）は、
> MVP後のIssueとして積み、別フェーズで対応する。

## 詳細フロー: US-02「思考記録を作成する」

> コアとなる価値提供ユースケース。**MVPはログイン済みユーザーのみ対象。**

### メインフロー（ログイン済み）

```
事前条件: ログイン済み

1. ユーザーがトップ画面を開く
2. 「記録する」ボタンを押す
3. 3コラムの入力フォームが表示される
   - 各コラムにプレースホルダー（入力例・ヒント）が表示されている
4. ユーザーが1つ以上のコラムを入力する
5. 「保存する」ボタンを押す
6. バリデーションを通過する
7. 記録が Supabase DB に保存される（user_id に紐づいて永続化）
8. 記録一覧画面（または保存した記録の詳細画面）に遷移する
9. 「記録しました」の成功フィードバックが表示される
```

### 代替フロー

| 番号 | 状況 | 動作 |
|------|------|------|
| A1 | すべてのコラムが空のまま「保存」を押した | バリデーションエラーを表示。「少なくとも1つ入力してください」などの案内 |
| A2 | 入力途中でキャンセル / 戻るを押した | 入力内容がある場合は確認ダイアログ（MVP はシンプルに確認なしでも可） |
| A3 | 保存時にネットワークエラーが発生した | エラーメッセージを表示。入力内容は画面に残す |

### バリデーションルール

| 項目 | 方針 |
|------|------|
| 必須コラム | **すべて任意**（書きやすさを優先。空でも保存可能） |
| 最低入力要件 | 1コラム以上に何らかの入力がある場合のみ保存可 |
| 文字数制限 | MVP は制限なし（将来的に上限設定を検討） |

## 将来対応ユースケースの概要メモ

> 以下は MVP 以降で GitHub Issues に起票する際の種として残す。

### US-01 / US-08: アカウントなしユーザー対応

- US-01: 未ログインのまま記録を作成・閲覧できる（localStorage に保存）
- US-08: アカウント作成時 or ログイン時に localStorage の記録を Supabase DB へ一括移行
- **実装上の注意点**:
  - アプリ全体で「ログイン状態かどうか」の分岐が必要になる
  - 移行処理のエラーハンドリング（ネットワークエラー時は localStorage を保持して再試行）
  - US-01 を先に実装し、US-08 はその後に対応する順序が望ましい

### US-10: 初回オンボーディング
- CBT / 3コラム法を知らないユーザー向けの初回説明フロー
- 「何を書けばいいか」「どんな効果があるか」を簡潔に案内
- スキップ可能にする（2回目以降は表示しない）

### US-11: 途中保存・下書き
- フォームの入力内容を localStorage に一時保存（認証状態に関わらず）
- 次回アクセス時に「書きかけの記録があります」と通知して復元できる

### US-12: 記録の振り返り・パターン発見
- 記録一覧にて状況や感情などでフィルタリング
- 同じような状況での記録を並べて比較できる

### US-14: アカウント削除・データ消去
- ユーザーがすべての記録と個人情報を完全に削除できる
- 削除前に確認ダイアログ（「この操作は取り消せません」）
- Clerk / Supabase 双方のデータを連携して削除

### US-15: 継続促進（ストリーク・実績）
- 連続記録日数の表示
- ストリークが途切れても気持ちよく再開できる設計

### US-17: AI によるバランス思考サジェスト
- 1・2コラムの入力をもとに Claude API が3コラム目の候補を提示
- あくまで「参考」として提示し、ユーザーが自由に編集できる
- プロンプト設計・倫理的配慮が必要
