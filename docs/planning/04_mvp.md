# MVP 定義

> ユースケースの詳細は [03_use-cases.md](./03_use-cases.md) を参照。

---

## CBT 3コラム法の仕様

### 採用する形式

**日本標準の3コラム法（ABC モデル）**

| 記号 | 役割 | 心理学的意味 |
|------|------|------------|
| A | 出来事（Activating Event） | 感情の引き金となった状況 |
| B | 自動思考（Beliefs / Automatic Thought） | 出来事に対する瞬間的な解釈・考え |
| C | 感情・行動（Consequences） | B によって引き起こされた感情や行動 |

> **参考**: 日本の3コラム法は「ABCの記録」に相当し、バランス思考（認知の書き換え）は5コラム法以降に登場する。
> 英語圏の "Triple Column Technique"（Burns）はBコラムに合理的反応が入る別形式であり、本アプリは日本標準に準拠する。

---

### コラムの表示定義

| コラム | 内部フィールド名 | 表示ラベル | 説明文（入力欄の上部） | 入力例 |
|--------|----------------|-----------|---------------------|--------|
| A | `situation` | できごと | どんなことがありましたか？ | 上司に「この資料、使えないな」と言われた |
| B | `thought` | どう捉えたか **MVPデプロイ以降でその他候補と再度比較する**　| 出来事が起きたとき、どんな考えが浮かんでいましたか？ | 「また自分がダメだった」「どうせ認めてもらえない」 |
| C | `emotion` | 感情 | どんな気持ちになりましたか？ | 落ち込んだ、悔しかった、不安になった |

### 入力ガイドの順序

フォームの**表示順はA→B→C**（できごと→思考・解釈→感情）で固定する。

ただし、B列（思考・解釈）は心理的に深い作業を必要とするため、**ヘルプテキストで段階的な記録を促す**。

```
できごと → 感情（書きやすいものから） → 思考・解釈（できれば）
```

> B列は「自己分析を伴う難しいステップ」であるため、空欄でも保存可能にする（バリデーションルール参照）。

---

### B. 自動思考のラベル・説明文候補

**MVP での採用: 「どう捉えたか」**

#### ラベル候補

| 候補 | 備考 |
|------|------|
| どう捉えたか | **MVP 採用** |
| 思考・解釈 | |
| 受け取り方・解釈 | |

#### 説明文候補

| 候補 | 備考 |
|------|------|
| 出来事が起きたとき、どんな考えが浮かんでいましたか？ | **MVP 採用** |
| 出来事が起きたとき、どんな考えが頭にありましたか？ | |

> MVP 以降、実際の利用状況をもとにラベル・説明文の表現を比較・検討して最終決定する。

---

### バリデーションルール

| 項目 | 仕様 |
|------|------|
| 必須コラム | なし（すべて任意） |
| 保存条件 | A・B・C のうち **1つ以上に入力**がある場合のみ保存可 |
| 文字数上限 | MVP は制限なし（将来検討） |
| 空欄の扱い | null として保存（空文字列ではなく null） |
| 確信度（`thought_confidence`） | 0〜100 の整数。未入力は null として保存 |
| 感情強度（`emotion_intensity`） | 0〜100 の整数。未入力は null として保存 |

## 除外する機能（MVP 以降）

| 機能 | 理由 |
|------|------|
| バランス思考コラム（5コラム法相当） | 自動思考の記録が定着してから段階的に追加 |
| アカウントなし記録（US-01） | 実装複雑度が高い。ログイン必須で MVP をシンプルに保つ |
| 記録引き継ぎ（US-08） | US-01 の前提なので同様に除外 |
| AI によるバランス思考サジェスト | 品質・倫理的配慮が必要。MVP 後に検討 |
| オンボーディング画面 | 入力例・プレースホルダーで代替 |
| タグ・フィルタリング | 記録数が蓄積されてから意味を持つ |

---

## 非機能要件

| 項目 | 方針 |
|------|------|
| レスポンシブ | モバイルファースト。SP / PC で使いやすいレイアウト |
| 初回記録時間 | 3分以内に1記録を完了できる UX を目標（KPI より） |
| アクセシビリティ | フォームのラベル・aria 属性を適切に設定（WCAG AA 相当を目安） |
| 医療的免責 | 「本アプリは医療行為・診断を行うものではありません」を明示 |
| 認証・セキュリティ | Clerk に委譲。SQL インジェクション・XSS は Next.js / Supabase の標準対策に準拠 |
| パフォーマンス | 一覧ページの初期表示は直近20件を取得（ページネーションは将来対応） |
