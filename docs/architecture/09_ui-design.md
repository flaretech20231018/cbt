# UI デザインガイド（横断ルール）

> 本ドキュメントはアプリ全体に適用される UI の横断ルールを定義する。
> 個別画面の仕様・遷移図は各機能の `docs/features/<feature>/spec.md` に記載する。

---

## デザイントークン

### カラーパレット

Tailwind CSS のデフォルトカラーをベースに、セマンティックな役割で定義する。
具体的なカラーコード値は `tailwind.config.ts` の `theme.extend.colors` で設定する。

| トークン名 | 役割 | Tailwind 目安 | 備考 |
|-----------|------|--------------|------|
| `primary` | 主要アクション（ボタン・リンク） | `blue-600` / `blue-700` | コントラスト比 4.5:1 以上を確保 |
| `primary-hover` | primary のホバー状態 | `blue-700` / `blue-800` | |
| `destructive` | 削除・危険な操作 | `red-600` | 確認ダイアログと併用 |
| `destructive-hover` | destructive のホバー状態 | `red-700` | |
| `bg-primary` | ページ背景 | `white` / `gray-50` | |
| `bg-secondary` | カード・セクション背景 | `gray-50` / `gray-100` | |
| `text-primary` | 本文テキスト | `gray-900` | |
| `text-secondary` | 補足テキスト・ラベル | `gray-500` / `gray-600` | |
| `border` | 境界線・区切り | `gray-200` / `gray-300` | |
| `success` | 成功メッセージ | `green-600` | |
| `warning` | 警告メッセージ | `amber-600` | |
| `error` | エラーメッセージ | `red-600` | |

> カラーの最終値は実装フェーズ（Phase 1）で `tailwind.config.ts` に定義する。
> ダークモードは MVP スコープ外。

### タイポグラフィ

| 要素 | クラス目安 | 備考 |
|------|-----------|------|
| ページタイトル（h1） | `text-2xl font-bold` | 各ページに 1 つ |
| セクション見出し（h2） | `text-xl font-semibold` | |
| 小見出し（h3） | `text-lg font-medium` | |
| 本文 | `text-base` (16px) | 最小フォントサイズ |
| 補足テキスト | `text-sm text-secondary` | ヘルプテキスト・タイムスタンプ等 |
| ラベル | `text-sm font-medium` | フォームラベル |

> フォントファミリーは Tailwind デフォルト（`font-sans`）を使用する。

### スペーシング

| 用途 | 値 | Tailwind クラス |
|------|-----|----------------|
| ページ外側余白 | 16px（SP）/ 24px（PC） | `px-4 md:px-6` |
| セクション間 | 24px〜32px | `space-y-6` / `space-y-8` |
| カード内余白 | 16px | `p-4` |
| フォーム要素間 | 16px〜20px | `space-y-4` / `space-y-5` |
| ボタン内余白 | 8px 16px | `py-2 px-4` |

### ブレークポイント

モバイルファーストで設計する。

| 名称 | 幅 | Tailwind prefix | 主な対象 |
|------|-----|----------------|---------|
| SP（デフォルト） | 〜639px | なし | スマートフォン |
| Tablet | 640px〜 | `sm:` | タブレット（必要に応じて） |
| PC | 768px〜 | `md:` | デスクトップ |

| レイアウト要素 | SP | PC |
|--------------|-----|-----|
| コンテンツ最大幅 | 100% | `max-w-2xl`（672px） |
| フォーム | 1カラム | 1カラム |
| 記録一覧 | 1カラム（カード縦並び） | 1カラム（カード縦並び） |

> MVP ではグリッドレイアウトは使わない。コンテンツ最大幅を制限してシンプルに保つ。

## 共通コンポーネントの振る舞い

### ボタン

| バリアント | 用途 | スタイル方針 |
|-----------|------|------------|
| Primary | 主要アクション（保存・作成） | `bg-primary text-white` |
| Secondary / Outline | 補助アクション（キャンセル・戻る） | `border border-border text-primary` |
| Destructive | 削除 | `bg-destructive text-white` |
| Ghost | インラインアクション | `text-primary hover:bg-gray-100` |

**共通ルール：**

- 送信中は `disabled` + スピナー表示。二重送信を防止する
- `disabled` 状態は `opacity-50 cursor-not-allowed`
- タッチターゲットは最小 44x44px（WCAG 2.2 Target Size）

### フォーム入力

| 要素 | ルール |
|------|-------|
| `<label>` | すべての入力欄に `<label>` を関連付ける（`htmlFor`） |
| プレースホルダー | 入力例を表示する。ラベルの代替にはしない |
| ヘルプテキスト | 入力欄の下に `text-sm text-secondary` で表示 |
| エラーメッセージ | 入力欄の下に `text-sm text-error` で表示。ヘルプテキストと排他 |
| フォーカス | `focus:ring-2 focus:ring-primary focus:outline-none` |

### カード

一覧画面の各記録を表示する共通パターン。

```
┌─────────────────────────────┐
│  2026-03-06                 │  ← 日付（text-sm text-secondary）
│  上司に「この資料、使えない  │  ← できごと冒頭（1〜2行で truncate）
│  な」と言われた              │
│                             │
│  落ち込んだ、悔しかった      │  ← 感情（text-sm text-secondary）
└─────────────────────────────┘
```

## エラー表示

### エラーの種類と表示方針

| 種類 | 発生場面 | 表示方法 | 表示位置 |
|------|---------|---------|---------|
| フィールドエラー | フォームバリデーション（Zod） | 入力欄直下にインラインメッセージ | 該当フィールドの下 |
| フォーム全体エラー | Server Action の戻り値 `{ error }` | アラートバナー | フォーム上部 |
| ページエラー | Server Component のデータ取得失敗 | `error.tsx` エラーバウンダリ | ページ全体 |
| 404 | 存在しない記録 | `not-found.tsx` | ページ全体 |

### フィールドエラー

```
ラベル
┌─────────────────────────────┐
│ 入力値                       │  ← border-error（赤い枠線）
└─────────────────────────────┘
⚠ エラーメッセージ               ← text-sm text-error
```

- `aria-invalid="true"` と `aria-describedby` でエラーメッセージを入力欄に関連付ける
- エラー発生時、最初のエラーフィールドにフォーカスを移動する

### フォーム全体エラー（アラートバナー）

```
┌─ ⚠ ─────────────────────────┐
│ 保存に失敗しました。          │  ← bg-red-50 text-error border-error
│ もう一度お試しください。      │
└─────────────────────────────┘
```

- `role="alert"` でスクリーンリーダーに通知する

### ページエラー（error.tsx）

```
┌─────────────────────────────┐
│                             │
│     エラーが発生しました      │
│                             │
│   ページの読み込みに失敗      │
│   しました。                 │
│                             │
│      [もう一度試す]          │  ← reset() を呼ぶ
│                             │
└─────────────────────────────┘
```

### 空状態（データなし）

```
┌─────────────────────────────┐
│                             │
│     まだ記録がありません      │
│                             │
│   最初の記録を作成して        │
│   みましょう                 │
│                             │
│      [記録を作成する]        │  ← /records/new へのリンク
│                             │
└─────────────────────────────┘
```

## ローディング表示

### 方針

| 場面 | 表示方法 | 実装 |
|------|---------|------|
| ページ遷移・データ取得 | スケルトン UI | `loading.tsx`（Suspense fallback） |
| フォーム送信中 | ボタン内スピナー + disabled | `useTransition` / `useFormStatus` |
| 削除処理中 | ボタン内スピナー + disabled | `useTransition` |

### スケルトン UI

`loading.tsx` でページ全体のスケルトンを表示する。コンテンツの配置を模したプレースホルダーで、レイアウトシフトを防ぐ。

```
┌─────────────────────────────┐
│  ████████ （タイトル）       │  ← bg-gray-200 animate-pulse rounded
│                             │
│  ┌───────────────────────┐  │
│  │ ██████████            │  │  ← カードのスケルトン
│  │ ████████████████      │  │
│  └───────────────────────┘  │
│  ┌───────────────────────┐  │
│  │ ██████████            │  │
│  │ ████████████████      │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
```

### ボタンのスピナー

```
送信中の状態:
┌────────────────────┐
│  ◌ 保存中...       │  ← opacity-80, cursor-not-allowed
└────────────────────┘
```

- ボタンテキストを「保存中...」等に変更し、SVG スピナーを左に配置する
- `disabled` 属性で二重送信を防止する

## アクセシビリティ（WCAG AA 準拠方針）

### 対応範囲

MVP では WCAG 2.1 AA を**目安**として設計する。完全準拠の第三者監査は MVP スコープ外。

### 必須対応項目

| カテゴリ | ルール |
|---------|-------|
| コントラスト | テキストと背景のコントラスト比 4.5:1 以上（大文字テキストは 3:1） |
| キーボード操作 | Tab キーですべてのインタラクティブ要素に到達できる。フォーカスインジケーターを視覚的に明示する |
| フォームラベル | すべての入力欄に `<label>` を関連付ける。視覚的に非表示にする場合は `sr-only` を使う |
| エラー通知 | フォームエラーは `aria-invalid` + `aria-describedby` で関連付ける。`role="alert"` でライブリージョンを使う |
| 画像の代替テキスト | `<img>` には必ず `alt` を設定する。装飾画像は `alt=""` |
| 見出し構造 | `h1` → `h2` → `h3` の順序を守る。レベルを飛ばさない |
| タッチターゲット | インタラクティブ要素の最小サイズは 44x44px |

### フォーカス管理

| 場面 | 振る舞い |
|------|---------|
| モーダル表示時 | フォーカスをモーダル内に閉じ込める（フォーカストラップ） |
| モーダル閉じた時 | フォーカスをトリガー要素に戻す |
| エラー発生時 | 最初のエラーフィールドにフォーカスを移動する |
| ページ遷移後 | メインコンテンツ領域の見出しにフォーカスする |

## 医療的免責の表示

### 表示ルール

| 項目 | 内容 |
|------|------|
| 表示テキスト | 「本アプリは医療行為・診断を行うものではありません。深刻な悩みがある場合は、専門家にご相談ください。」 |
| 表示場所 | 共通フッター（`components/layout/Footer.tsx`） |
| 表示条件 | すべてのページで常時表示 |
| スタイル | `text-xs text-secondary` で控えめに表示 |

### 実装方針

- Root Layout（`app/layout.tsx`）に `<Footer />` を配置し、全ページに自動適用する
- 免責テキストはハードコードで問題ない（多言語対応時に `i18n` 化する）

## トースト・通知（MVP では見送り）

MVP ではトースト通知は実装しない。Server Action の結果は以下の方法で伝える。

| 結果 | 表示方法 |
|------|---------|
| 成功 | ページ遷移で暗黙的に伝える（例：作成後 → 一覧へ遷移） |
| エラー | フォーム内のアラートバナーで表示 |

> トースト通知は Phase 3（コア機能実装）の SDD サイクル内で検討する。
