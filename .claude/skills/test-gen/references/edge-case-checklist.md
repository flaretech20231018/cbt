# エッジケースチェックリスト

テストケース列挙時に、パラメータの型に応じて該当セクションを照合する。
各項目の「狙い」は、そのテストが防止するバグの種類を示す。

---

## 文字列

| チェック項目 | テスト値 | 狙い |
|-------------|---------|------|
| null/undefined | `null`, `undefined` | null参照エラー |
| 空文字列 | `""` | 空値処理 |
| 空白のみ | `"   "`, `"\t\n"` | trim処理漏れ |
| 長さ1 | `"a"` | 最小有効長 |
| 最大長超過 | スキーマのmax+1文字 | バリデーション境界 |
| 最大長ちょうど | スキーマのmax文字 | off-by-one |
| 先頭/末尾空白 | `"  alice  "` | trim漏れ |
| HTMLタグ | `"<script>alert(1)</script>"` | XSS（CWE-79） |
| SQLメタ文字 | `"'; DROP TABLE--"` | SQLインジェクション（CWE-89） |
| Unicode絵文字 | `"テスト😀入力"` | エンコーディング |
| RTL文字 | アラビア語文字 | レイアウト崩れ |
| ゼロ幅スペース | `"\u200B"` | 見えない文字 |
| 改行含有 | `"line1\nline2"` | 複数行処理 |

## 数値

| チェック項目 | テスト値 | 狙い |
|-------------|---------|------|
| ゼロ | `0` | ゼロ除算、falsy判定 |
| 負数 | `-1` | 符号想定漏れ |
| 小数 | `0.1`, `99.999` | 整数前提の処理 |
| 最小値 | スキーマのmin | 境界値 |
| 最小値-1 | スキーマのmin - 1 | off-by-one |
| 最大値 | スキーマのmax | 境界値 |
| 最大値+1 | スキーマのmax + 1 | off-by-one |
| NaN | `NaN` | 数値演算の伝播 |
| Infinity | `Infinity` | 比較・演算の異常 |
| 浮動小数点精度 | `0.1 + 0.2` | IEEE 754 |
| 文字列の数値 | `"42"` | 型強制 |

## 配列・リスト

| チェック項目 | テスト値 | 狙い |
|-------------|---------|------|
| 空配列 | `[]` | length===0の処理 |
| 要素1つ | `[x]` | 最小有効セット |
| null要素含有 | `[1, null, 3]` | null伝播 |
| 重複要素 | `[1, 1, 1]` | 一意性前提の処理 |
| 大量要素 | 10000要素 | パフォーマンス・ページネーション |
| ページネーション境界 | page 0, page 1, 最終ページ, 最終ページ+1 | off-by-one |

## 日付・時刻

| チェック項目 | テスト値 | 狙い |
|-------------|---------|------|
| 閏年2月29日 | `2024-02-29` | 閏年処理 |
| 閏年でない2月29日 | `2023-02-29` | 不正日付 |
| 月末 | `2024-01-31` | 月末処理 |
| 年末年始 | `2024-12-31` → `2025-01-01` | 年跨ぎ |
| Unix epoch | `1970-01-01T00:00:00Z` | 初期値との誤判定 |
| 遠い未来 | `2099-12-31` | オーバーフロー |
| 無効な日付文字列 | `"not-a-date"` | パース失敗 |
| タイムゾーン | UTC vs JST（+9h） | サーバー/クライアント差異 |
| DST遷移 | 春進み（2:00AM→3:00AM）、秋戻り | 時刻の飛び・重複 |

出典: Noah Sussman "Falsehoods Programmers Believe About Time", EA Leap Day 2024バグ事例

## 真偽値・フラグ

| チェック項目 | テスト値 | 狙い |
|-------------|---------|------|
| falsy値 | `false`, `0`, `""`, `null`, `undefined` | JS truthy/falsy混同 |
| 文字列"false" | `"false"` | truthyだが意味的にfalse |
| 文字列"0" | `"0"` | truthyだが意味的にゼロ |

## オブジェクト・JSON

| チェック項目 | テスト値 | 狙い |
|-------------|---------|------|
| 空オブジェクト | `{}` | プロパティアクセスエラー |
| optionalフィールド欠損 | キーごと存在しない | undefined判定 |
| ネストnull | `{ address: null }` → `.address.city` | null参照 |
| 余分なフィールド | 未定義キーを含む | ホワイトリスト漏れ |
| 不正なJSON文字列 | `"{ invalid"` | JSON.parse失敗 |

## 仕様として抜け漏れやすい領域（Falsehoods Programmers Believe）

### 人名

- 全員がファーストネーム/ラストネームを持つわけではない
- 名前にアポストロフィ（O'Brien）、ハイフン、スペースを含む場合がある
- 名前はASCII文字だけで表現できるとは限らない
- 「Mr. Null」という名前の人が実在する

### メールアドレス

- `@`を1つだけ含むとは限らない
- ローカルパートに`+`や`.`を含むのはRFC上有効（`user+tag@example.com`）

### 住所

- 全ての住所が「番地、通り名、市」の構造を持つわけではない
- 郵便番号の形式は国によって異なる

## テスト対象の優先度ガイド（統計的根拠）

Step 3での列挙優先度。上位ほどバグ発見確率が高い。

1. **エラーハンドリングパス** — Yuan et al. OSDI 2014: カタストロフィック障害の92%が不正なエラー処理に起因。77%がユニットテストで再現可能
2. **null/undefined入力** — Li et al. ICSE 2013: Bingの障害の84.5%がデータ問題。修正の93%がnullチェック・デフォルト値の追加のみ
3. **認証・認可** — CWE Top 25 2024/2025: アクセス制御関連CWEが7つランクイン。OWASP Top 10で第1位
4. **入力バリデーション** — CWE Top 25 2024: 入力処理不備が全スコアの60%を占める
5. **境界値** — BVA: 欠陥は入力範囲の端に集中する。off-by-oneエラー（CWE-193）はセキュリティ脆弱性にも直結
